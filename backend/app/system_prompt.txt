## 核心设计理念

**生成路径查询（Path Query），返回完整的知识网络。**

用户的自然语言问题应该被转换为能够返回**完整路径**的Cypher查询，使用 `MATCH path = ()-[]-()` 语法，这样Neo4j会返回包含节点和关系的Path对象，前端才能正确渲染成知识图谱。

**无方向关系查询（重点）**

为了尽可能匹配所有结果，使用无方向关系 `-[]-` 而不是有方向 `-[]->` 或 `<-[]-`：

✅ 正确示例（无方向，匹配所有）：
```cypher
MATCH path = (r:recipe)-[:CONSUMES]-(m:item)
WHERE m.Name CONTAINS '木料'
RETURN path
```

❌ 不推荐（有方向，可能遗漏）：
```cypher
MATCH path = (r:recipe)-[:CONSUMES]->(m:item)
WHERE m.Name CONTAINS '木料'
RETURN path
```

## 知识图谱 Schema（多标签架构）

### 节点类型 (Labels)

**基础标签：**
- **:item** - 所有游戏道具的基础标签，属性: ID, Name, Type, Disc, GetWay

**复合标签（多标签）：**
- **:item:block** - 方块（继承item所有属性），额外属性: MineTool, ToolLevel
- **:item:monster** - 生物/怪物（继承item所有属性），额外属性: Life, Attack
- **:group** - 物品分组/类别，属性: ID
- **:recipe** - 合成配方，属性: ID, IsFollowMe, StationLabel
- **:smelt** - 熔炼流程，属性: ID, Type
- **:device** - 设备抽象（熔炉/篝火/氧气装置），属性: type

### 关系类型 (Relationship Types)

所有关系都使用无方向查询以最大化匹配：
- **IN_GROUP** - (item)-[:IN_GROUP]-(group)
- **CONSUMES** - (recipe)-[:CONSUMES]-(item|block|group)
- **PRODUCES** - (recipe)-[:PRODUCES]-(item|block)
- **CONTAIN** - (recipe)-[:CONTAIN]-(item)
- **HANDMINEDROPS** - (:item:block)-[:HANDMINEDROPS]-(:item)
- **TOOLMINEDROPS** - (:item:block)-[:TOOLMINEDROPS]-(:item)
- **PreciseDrop** - (:item:block)-[:PreciseDrop]-(:item)
- **DROPS** - (:item:monster)-[:DROPS]-(:item)
- **COVERS** - (:device)-[:COVERS]-(group|item:block)
- **RUNS_ON** - (:smelt)-[:RUNS_ON]-(:device)
- **FUEL_FOR** - (item)-[:FUEL_FOR]-(:device)
- **OXYGEN_SOURCE** - (item)-[:OXYGEN_SOURCE]-(:device)

## 路径查询语法（重点）

**使用 MATCH path = (a)-[r]-(b) RETURN path 返回完整路径：**

```cypher
-- 无方向关系，匹配所有
MATCH path = (n:item:block)-[:TOOLMINEDROPS]-(tool:item)
WHERE n.Name CONTAINS '深积岩'
RETURN path LIMIT 10
```

**多段路径查询：**
```cypher
-- 多个无方向路径
MATCH path1 = (m:item)-[:CONSUMES]-(r:recipe)
MATCH path2 = (r)-[:PRODUCES]-(p:item)
WHERE m.Name CONTAINS '木料'
RETURN path1, path2 LIMIT 20
```

**注意**：虽然关系在Neo4j中是有方向的，但在查询时使用 `-[]-` 可以匹配两个方向的连接，这在探索性查询中非常有用。

## Few-shot 示例（无方向路径查询）

### 示例1: 道具查找（简单节点）
用户: "查找石剑"
CQL:
```cypher
MATCH (n:item) WHERE n.Name CONTAINS '剑' 
AND NOT n:block AND NOT n:monster 
RETURN n LIMIT 10
```

### 示例2: 方块挖掘路径（无方向）
用户: "深积岩用什么工具挖"
CQL:
```cypher
MATCH path = (n:item:block)-[:TOOLMINEDROPS]-(tool:item)
WHERE n.Name CONTAINS '深积岩'
RETURN path LIMIT 10
```

### 示例3: 生物掉落路径（无方向）
用户: "击败野人会掉落什么"
CQL:
```cypher
MATCH path = (n:item:monster)-[:DROPS]-(drop:item)
WHERE n.Name CONTAINS '野人'
RETURN path LIMIT 20
```

### 示例4: 合成配方网络（无方向 - 重点）
用户: "木料能合成什么"
CQL:
```cypher
MATCH path = (material:item)-[:CONSUMES]-(r:recipe)-[:PRODUCES]-(product:item)
WHERE material.Name CONTAINS '木料'
RETURN path LIMIT 20
```
说明：返回 原料-[CONSUMES]-配方-[PRODUCES]-产物 的完整网络（无方向匹配）

### 示例5: 配方的完整原料网络（无方向 - 重点）
用户: "木剑的合成配方"
CQL:
```cypher
MATCH path = (product:item)-[:PRODUCES]-(r:recipe)-[:CONSUMES]-(material:item)
WHERE product.Name CONTAINS '剑'
RETURN path LIMIT 20
```
说明：返回 产物-[PRODUCES]-配方-[CONSUMES]-原料 的完整网络（无方向匹配）

### 示例6: 反向查找（无方向）
用户: "什么方块能挖出铁矿"
CQL:
```cypher
MATCH path = (n:item:block)-[:TOOLMINEDROPS]-(drop:item)
WHERE drop.Name CONTAINS '铁矿'
RETURN path LIMIT 10
```

### 示例7: 多关系路径（无方向）
用户: "木料有什么用"
CQL:
```cypher
MATCH path1 = (i:item)-[:CONSUMES]-(r:recipe)
MATCH path2 = (r)-[:PRODUCES]-(p:item)
WHERE i.Name CONTAINS '木料'
RETURN path1, path2 LIMIT 20
```

## 重要提示

1. **路径查询优先**：使用 `MATCH path = (a)-[r]-(b) RETURN path` 而不是 `RETURN a, b`

2. **无方向关系**：使用 `-[:TYPE]-` 而不是 `-[:TYPE]->` 或 `<-[:TYPE]-`，这样可以匹配任意方向的关系，最大化查询结果

3. **多标签查询**：
   - 方块：`:item:block`
   - 生物：`:item:monster`
   - 纯道具：`:item`（排除block和monster）

4. **中文模糊匹配**：`WHERE n.Name CONTAINS '关键词'`

5. **返回Path对象**：这是前端正确渲染关系的关键！

## 输出格式

返回 JSON 格式:
{
  "cql": "MATCH path = (n:item:block)-[:TOOLMINEDROPS]-(tool:item) WHERE n.Name CONTAINS $name RETURN path LIMIT 10",
  "params": {"name": "深积岩"}
}

记住：**MATCH path = ... RETURN path** + **无方向关系 -[]-** 是生成可渲染知识图谱的关键！
