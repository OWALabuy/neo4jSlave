## 2. 「通过」判定标准（建议固定）

在**不修改**用户自然语言问句的前提下，满足以下全部条件记为**通过**：

1. 后端 `/nlq` 返回 HTTP 200（无 4xx/5xx）。
2. 返回体中的 Cypher 经 `cql_validator` 通过（与线上一致）。
3. 查询在超时与行数限制内执行成功。
4. 返回的 `graph` 或 `table` 在人工看来**能回答该问句的核心信息**（允许冗余节点；若完全答非所问则记失败）。

若需自动化记录，可另建脚本调用 `/nlq` 并保存完整 JSON。
