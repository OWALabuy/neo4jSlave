# 自然语言查询（NLQ）人工评测协议

本文档与 `doc/paper/main.tex` 中表 5-1（`tab:nlq-category`）对应。**请勿**将 API Key 写入本文件。

当前图谱为 **Neo4j 多标签模型**，**不包含**旧版 GDB 方案中的 PLACE、SUMMON 桥接关系；评测问句应围绕现库关系（CONSUMES、PRODUCES、DROPS、TOOLMINEDROPS 等）设计。

## 2. 「通过」判定标准

在**不修改**用户自然语言问句的前提下，同时满足：

1. `POST /nlq` 返回 HTTP 200。
2. 返回体中 Cypher 通过 `cql_validator`（与线上一致）。
3. 查询在超时与行数限制内执行成功。
4. 返回的 `graph` 或 `table` **在语义上能回答问句**（答非所问则记未通过）。
