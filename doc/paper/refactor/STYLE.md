# 风格章程（STYLE.md）

> 本文件是论文重构工作的风格基准。抽取阶段和重写阶段的所有 agent 在开始任何工作之前必须先读完本文件，并以一句话向使用者报告自己理解到的三条最重要原则。

---

## 一、目标读者

中国某理工类大学计算机与信息技术学院的本科毕业论文答辩老师。具有计算机本科以上专业背景，但**不一定熟悉 Neo4j、Cypher、ECharts、FastAPI 这些具体工程技术**。

读者期望读到的是：清晰的中文学术散文，逻辑递进有秩序，技术名词在第一次出现时被恰当引出。

读者**不期望**读到的是：

- 直译式的英语句法
- 用括号给每个名词补一段定义
- 一段话里同时讲架构、安全、部署三件事
- 生造的"……性"名词
- 复古文言色彩的连接词

---

## 二、正面范例（金标准）

下面这一段是导师亲自改写的版本，**重写阶段输出的语言风格应当与之同档**：

> 知识图谱以图结构表示实体及实体间的关系，用以展示数据库中隐含的知识，在知识管理、智能问答与推荐等任务中应用较广。在属性图数据库中，若要查明实体之间的关系，通常需要使用专业的数据库查询语言，使用者需掌握数据库查询语言的使用规则及专业术语，且查询结果以表格方式呈现。这为普通用户使用知识图谱设置了技术门槛：一方面是要专门学习数据库查询语言的使用规则，另一方面难以直观从表格中获得到有效的信息。本文设计基于自然语言的知识图谱查询与可视化系统，使用户能够直接使用自然语言与系统交互，并通过浏览器直观浏览图形化的知识图谱。

请观察这一段为什么读起来顺畅：

1. **概念铺设有秩序**：先讲知识图谱是什么、做什么用，再引出"使用门槛"，最后给出本文方案。读者每读一句，前面都已经准备好了背景。
2. **不直接抛专有术语**：用"专业的数据库查询语言"取代"Cypher"，用"图形化的知识图谱"取代"ECharts 关系图"。具体技术名词放到正文第二章再正式介绍。
3. **每句都有清晰的主谓宾**：没有长定语前置，没有把主语压到末尾。
4. **整段没有括号**：所有信息都通过正常句式给出，不靠括号塞补充说明。
5. **没有翻译腔连接词**：没有"上述"、"据此"、"凡"、"宜"、"亦"、"所谓"。
6. **段落主题单一**：这一段只做"提出问题—说明门槛—给出方案"这一件事。

## 三、反面模式清单

以下每一条**反例都来自学生原稿**，正例是改写建议。重写时务必避免反例的句式与遣词。

### 模式 1：英译腔的"构成……门槛"

- 反例：上述步骤构成面向终端用户的使用门槛。
- 正例：这就给非专业用户造成了使用上的门槛。

### 模式 2：用括号给术语塞定义

- 反例：基于 ECharts（一种在浏览器中绘制图表的常用开源库）的点边关系图。
- 正例：以浏览器中可交互的关系图形式展示（具体使用 Apache ECharts 实现）。
- 更佳：直接在前一段或本段开头用一句完整句子引出 ECharts，本段就不再加括号。

### 模式 3：未铺垫的术语直接出现

- 反例：通常须书写 Cypher。
- 正例：通常需要使用专业的数据库查询语言；本文采用 Neo4j 图数据库，其查询语言称为 Cypher。

### 模式 4：长定语前置、主语后置

- 反例：本研究面向希望在浏览器中浏览领域知识、但不打算为偶尔几次查询而系统学习 Cypher 与专用客户端的读者。
- 正例：本研究面向这样一类读者：他们希望在浏览器中浏览领域知识，但不愿意为偶尔的查询专门学习数据库查询语言。

### 模式 5：一段塞多件事

- 反例：在一段摘要里同时讲 B/S 架构、提示词与 Schema、只读 Cypher 的定义、安全校验、可视化形式、环境变量配置——六件不同抽象层级的事。
- 正例：摘要按"为什么做—做了什么—怎么验证—结果如何"四段写，每段只讲一件事。

### 模式 6：生造"……性"名词

- 反例：可观测性、可感知性、可外推性、可外推范围。
- 正例：可观察、便于读者核对、可推广到其他数据集、适用范围。

### 模式 7：复古文言色彩的连接词

- **禁用**：上述、据此、凡、宜、亦、所谓、即（用作括号内引导时）、该（作代词时）。
- 替换示例：
  - "上述步骤" → "前面这几步"
  - "据此本文实现……" → "基于这一思路，本文实现了……"
  - "凡进入 Neo4j 的语句" → "任何进入 Neo4j 的语句"
  - "宜将允许列表收紧" → "应当把允许列表收紧"
  - "亦" → "也"

### 模式 8：动词被名词吞掉

- 反例：进行测试、完成实现、开展讨论、施加约束。
- 正例：测试、实现、讨论、约束。

### 模式 9：括号里塞主张性内容

- 反例：通过环境变量配置（即在操作系统或部署环境中设置的命名参数，不写入论文正文）。
- 正例：通过环境变量配置；具体含义见第 X 节。

括号只用于：补充一个**纯客观、纯简短**的说明（例如缩写全称："表述性状态转移（REST）"）。一旦括号里出现"即……"、"指……"或带有判断的句子，就应当拆出去做正文或脚注。

### 模式 10：不必要的被动腔

- 反例：21 条问句中 17 条被判为可接受。
- 正例：21 条问句中有 17 条判定为可接受。

## 四、硬约束（数字化）

下列条件应当机械式遵守。重写阶段每段输出后必须自检并报告。

- 单段不超过 **250 字**。
- 单句不超过 **50 字**。
- 单段中**括号注释最多 1 处**，且括号内不超过 15 字。
- 段落必须有**清晰的主题句**（一般为段首句）。
- 所有 LaTeX 命令（`\cite{}`、`\ref{}`、`\label{}`、`\texttt{}`、`\url{}`、`\emph{}` 等）**原样保留**，不得删除、改名或更换序号。
- 表格、代码块（`lstlisting`、`verbatim`）、图片占位符（`\includegraphics`、`\caption`、`\label`）**原封不动**保留位置。
- 所有反面模式清单中的**禁用词**不得出现。

## 五、术语首次出现处理规范

整篇论文必须遵守"术语首次铺设秩序"：

- **核心术语**（Cypher、Neo4j、ECharts、FastAPI、知识图谱、属性图、大语言模型等）：在**第一次出现**时用一整句话引出，给出中文意思与英文原文，可附文献引用。后续直接使用即可。
- **普通工程概念**（环境变量、跨域、虚拟环境、HTTP 接口）：假定读者具备本科基础，不用解释。
- **课题项目名 neo4jSlave**：第一次出现时必须明确其为本系统的代号（例如"本文实现的系统命名为 neo4jSlave，下文统一以该名称指代"）。

抽取阶段如果发现某术语在原文中"首次出现没有铺垫"，应在节级核对中提出。

## 七、给 agent 的最后提醒

1. 你**不是在润色一份已存在的文字**，你是在根据一组事实要点**从零写一段中文学术散文**。原文的句式、词汇、连接习惯都和你无关。
2. 你的目标读者是**有耐心读完一段话**的中国本科答辩老师，不是搜索引擎，不是审稿人，也不是同行。请用学院派的简洁中文，不要堆砌术语证明自己专业。
3. 一旦你发现自己想写"上述"、"据此"、"亦"、"凡"——立即停下，换一种说法。
4. 如果一段话写完发现需要 3 处以上括号才能解释清楚，那是写作思路出了问题，不是括号本身的问题。重新想这一段要讲什么。
