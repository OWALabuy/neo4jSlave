# 论文重构操作手册

> 本目录为论文文字重构所用的提示词模板与流程说明。整套流程的设计目的：让 AI agent 帮助你把论文从"翻译腔学术垃圾"改成"读得通的中文学术散文"，且保证修改后的论文不会被 AI 默认风格反复污染。

---

## 一、为什么是两步走

直接让 agent"改写一段话"看起来简单，实际上效果很差。原因是：

> AI 模型在生成下一个字时，**最近上下文里的文字对它的句法选择影响极大**。所以一旦你把原文塞给它，让它"重写得更好",它的输出会保留大量原文的句式痕迹（"上述……构成……门槛"这种翻译腔结构会顽固地复现）。约束词、风格指令在这种情况下的效果都很有限。

**唯一彻底的办法是：让重写阶段的 agent 物理上看不到原文。**

所以本流程把工作切成两个独立会话：

1. **第一步（抽取阶段）**：在会话 A 里，把原文压缩成纯命题列表（事实要点）。这一步不写新文字，只做信息抽取。
2. **第二步（重写阶段）**：**新开会话 B**，agent 此前从未见过原文。给它风格章程 + 第一步抽出的要点，让它从零写一段新文字。

这样原文的句式就完全无法污染重写结果，因为 agent 根本没看到过那些句式。

## 二、文件清单

本目录下的文件与各自作用：

| 文件 | 作用 | 谁读它 |
|------|------|--------|
| `STYLE.md` | 风格章程，定义目标读者、正反范例、硬约束 | 抽取 agent 与重写 agent 都要先读 |
| `EXTRACT_PROMPT.md` | 抽取阶段的提示词模板 | 第一步会话 A 的 agent |
| `REWRITE_PROMPT.md` | 重写阶段的提示词模板 | 第二步会话 B 的 agent |
| `README.md`（本文件） | 操作手册 | 你 |
| `sec_X.facts.md` | 第一步产出的要点文件 | 由你保存，作为第二步输入 |

## 三、章节分级

不是所有章节都要做完整的两步重构。按以下分级处理：

### A 级（必须完整两步重构）

这一级的章节是答辩老师最先看、最容易暴露文字问题的部分：

- 摘要（中英文）
- 1.1 研究背景
- 1.3 问题描述与挑战
- 1.4 课题意义
- 1.5 本文主要工作
- 6.1 测试目的与方法
- 结论（包括主要工作、可核对要点、不足与展望）

### B 级（润色档，单步即可）

技术性章节内容相对固定，做一遍语言润色，不打乱结构：

- 1.2 国内外研究现状（三个子节）
- 第 2 章 系统开发环境与技术选型
- 第 3 章 系统分析中除去问题描述以外的部分
- 第 4 章 系统设计的四个模块小节
- 第 5 章 详细设计与实现
- 6.2 分模块测试

润色档的提示词是 `REWRITE_PROMPT.md` 的简化版（见第六节）。

### C 级（不动档）

不要碰：

- 所有表格内容
- 所有 `lstlisting` / `verbatim` 代码块
- 所有图片占位符与图注
- 参考文献
- 致谢（致谢是全文最像人话的部分，反而是好的）
- `thesis-settings.tex` 与一切 LaTeX 配置

## 四、A 级章节的标准操作流程

以下用"摘要"为例。其他 A 级章节同理，只是替换原文。

### Step 1：开启抽取会话（会话 A）

打开你的 AI agent（Cursor / Claude / ChatGPT 任意一个），新建一个空白会话。

**消息 1**（粘贴 `STYLE.md` 全文）：

```
请阅读以下风格章程，读完用一句话告诉我你接下来抽取要点时最关注的三件事是什么。

（粘贴 STYLE.md 全文）
```

**消息 2**（粘贴 `EXTRACT_PROMPT.md` 全文）：

```
请阅读以下抽取阶段提示词，确认你理解任务后回复"准备好了"。

（粘贴 EXTRACT_PROMPT.md 全文）
```

**消息 3**（粘贴一节原文 LaTeX）：

```
请按提示词要求抽取以下小节。完成后，给出节级核对结果。

（粘贴 main.tex 中本节的原始 LaTeX，从 \begin{abstract} 到 \end{abstract}，包含所有 \cite 等命令）
```

agent 会输出一份段落要点清单 + 节级核对。

### Step 2：人工复核

读一遍 agent 给的要点：

- 是否有要点其实是修辞句被硬抽出来的（"形成可运行链路"——这种空话应当丢弃）
- 是否有原文重要事实被漏掉
- 节级核对里 agent 提的"建议补充"——是否合理，要补的话补什么

把最终版（你认可的版本）保存为：

```
doc/paper/refactor/sec_abstract.facts.md
```

如果 agent 的要点你觉得有问题，**直接在会话里要求它修正**（"请把第 3 段的要点 3.2 改成更干净的命题"），不要省这一步——下一阶段的写作完全依赖这份要点的质量。

### Step 3：**新开会话**（会话 B）

**关键：必须新开。不要继续刚才的会话。**

为什么必须新开：刚才那个会话里 agent 已经看过了原文。如果你继续在它里面让它重写，原文的句式痕迹仍然在它的上下文里。这样两步走就白做了。

### Step 4：重写

在新会话里：

**消息 1**（粘贴 `STYLE.md` 全文）：

```
请阅读以下风格章程，读完用一句话告诉我你接下来写作时最优先遵守的三条原则是什么。

（粘贴 STYLE.md 全文）
```

**消息 2**（粘贴 `REWRITE_PROMPT.md` 全文）：

```
请阅读以下重写阶段提示词。注意第一条纪律：你不能向我索要原文。

（粘贴 REWRITE_PROMPT.md 全文）
```

**消息 3**（要点 + 元数据）：

```
请按提示词要求重写一节。先输出段落规划，等我确认后再开始写。

【节标题】摘要
【上一节】（无，是论文最前面）
【下一节】英文摘要 Abstract
【本节论证目的】用一段文字让答辩老师在 30 秒内明白：为什么做这个课题、做了什么、怎么验证、结果怎样。

【要点】
（粘贴 sec_abstract.facts.md 全部内容）
```

agent 会先回一个段落规划。

### Step 5：确认规划

如果规划合理（每段主题清楚、要点都被分配、段落顺序自然），回"按此规划开始写"。如果不合理，让它调整。

### Step 6：逐段交付

agent 一段一段写，每段附自检表。看自检表：

- 字数 ≤ 250、最长句 ≤ 50、括号 ≤ 1、无禁用词、无要点外事实、LaTeX 标记齐全——通过，让它继续下一段。
- 任一项不达标——让它重写：

```
你刚才输出的段落自检显示触发了禁用词"上述"，请回到规划，重新写这一段。
```

### Step 7：节级核对 + 落盘

所有段落写完后 agent 会做节级核对。看一遍是否有要点没覆盖。

把整节输出贴回 `main.tex` 的对应位置。如果有 `\cite` 等命令位置不对，手动调一下。

### Step 8：编译检查

```
xelatex main.tex
```

确认没有编译报错（缺花括号、缺 `\cite` 引用等）。如果有，对照原文修复。

## 五、B 级章节的简化流程

B 级章节做单步润色即可。打开一个新会话：

**消息 1**：粘贴 `STYLE.md`。

**消息 2**：

```
我会给你一节 LaTeX 论文原文，请按 STYLE.md 的风格要求做语言润色。要求：

1. 不打乱段落结构、不改变论证顺序、不增删事实。
2. 修复 STYLE.md 中列出的所有反面模式（翻译腔、括号塞定义、禁用词、生造"……性"名词、动词被名词吞掉等）。
3. 所有 LaTeX 标记原样保留。
4. 表格、代码块、图片占位符不动。
5. 每段输出后附自检表，发现自检不达标的段落自行重写。

如果发现某段需要补充内容才能通顺，在 【备注】 区块提出，不在正文里编造。
```

**消息 3**：粘贴一节原文。

收到输出后，看自检表，落盘。

## 六、三句固定话术

复制粘贴用，不需要改。

### 启动话术（每次新会话第一句）

```
请先阅读以下风格章程，确认理解后用一句话告诉我你接下来要遵守的最重要的三条原则。

（粘贴 STYLE.md）
```

### 抽取话术

```
请按 EXTRACT_PROMPT.md 的要求抽取以下小节的事实要点。完成后给出节级核对结果。

（粘贴 EXTRACT_PROMPT.md）

【原文】
（粘贴一节 main.tex）
```

### 重写话术

```
请按 REWRITE_PROMPT.md 的要求重写一节。先给段落规划，等我确认后再开始写。

（粘贴 REWRITE_PROMPT.md）

【节标题】
【上一节】
【下一节】
【本节论证目的】

【要点】
（粘贴 sec_X.facts.md）
```

### 回退话术（agent 输出不合格时）

```
你的第 N 段触发了 X（具体说明：长句超 50 字 / 括号 > 1 / 出现禁用词"上述" / 引入要点外事实）。请回到要点重写该段，输出前自检并把自检表更新。
```

## 七、纪律提醒

整套流程能否生效，取决于以下纪律是否严格执行：

1. **抽取与重写必须用不同会话**——这是核心，不能省。
2. **不要把 main.tex 原文丢给重写阶段的 agent**——丢了等于前功尽弃。
3. **每段都看自检表**——agent 自己会填，但它有时填得马虎，你扫一眼就能发现"字数 280"或"括号 2 处"。
4. **看到"上述"、"据此"、"亦"、"凡"立即让它重写**——这些词在 STYLE.md 里禁用，但 agent 可能漏检，你要做最后一道关。
5. **要点是不是真的都被覆盖了**——重写阶段的 agent 偶尔会漏掉一两条要点，节级核对时核对一下。
6. **编译能过**——LaTeX 命令一旦丢了，编译会出红字。每改完一节就编译一次，错就当场修。

## 八、时间预算（参考）

按 A 级章节算，单节约需 30–60 分钟：

- 抽取：10–15 分钟
- 人工复核要点：5–10 分钟
- 重写规划与逐段写作：15–25 分钟
- 自检 + 落盘 + 编译：5–10 分钟

A 级章节合计约 6 节，B 级章节做润色每节约 10–15 分钟（共约 15 节）。**整个工作量约 8–12 小时**，建议分两天做完，避免疲劳后审校失效。

---

## 九、最后一句

工具不能替代你自己的判断。每段输出你都要扫一眼，哪怕只看自检表的一行——只要你看了，绝大多数低级错误就过不了你这关。如果你完全不看，agent 默认行为又会把论文带回老路上。
