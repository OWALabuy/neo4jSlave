	\hspace{1.5em}Knowledge graphs, as semantic networks, can describe entities and their relationships in graph structure, and have wide applications in knowledge management, intelligent question answering, and recommendation systems. However, traditional knowledge graph queries require users to master query languages such as Cypher, which presents certain barriers for non-technical users.

	\hspace{1.5em}This paper designs and implements a natural language-based knowledge graph query and visualization system (neo4jSlave). The system adopts B/S architecture, builds backend services with FastAPI, uses large language models to convert natural language to Cypher queries, and utilizes ECharts for visualization of query results. The system features read-only security restrictions, parameterized queries, and multi-data source adaptation, lowering the barrier to using knowledge graphs.

\section{系统概述}
\subsection{概述}

知识图谱（Knowledge Graph）是一种用于描述实体、概念及其之间关系的语义网络，采用图结构对知识进行建模和存储。近年来，随着大数据和人工智能技术的发展，知识图谱在智能搜索、推荐系统、金融风控、生物医学等领域得到了广泛应用。Neo4j作为目前最流行的图数据库之一，采用属性图模型存储数据，使用Cypher查询语言进行数据操作，为知识图谱的存储和查询提供了高效的支持。

然而，Cypher查询语言具有一定的学习门槛，非技术背景的用户难以直接编写查询语句。为了降低知识图谱的使用门槛，让普通用户也能方便地查询和探索知识图谱中的数据，本文设计并实现了一个基于自然语言的知识图谱查询与可视化系统（neo4jSlave）。该系统允许用户以自然语言的形式输入查询需求，系统自动将其转换为Cypher查询语句并执行，最终以直观的图谱形式展示查询结果。

\subsection{课题意义}

现有的知识图谱查询方式主要存在以下问题：

（1）\textbf{学习门槛高}：用户需要掌握Cypher、SPARQL等图查询语言的语法，对非技术用户不够友好；

（2）\textbf{查询效率低}：复杂的图查询语句编写耗时，容易出错；

（3）\textbf{结果不直观}：传统的表格形式难以直观展示图结构数据中的关联关系。

基于上述问题，本系统利用大语言模型（LLM）的自然语言理解能力，实现自然语言到Cypher查询的自动转换，结合ECharts图可视化库，为用户提供直观、便捷的知识图谱查询体验。该系统在教育、企业知识管理、数据分析等场景具有重要的应用价值。

\subsection{主要内容}

本文从系统需求分析、架构设计、详细实现、系统测试等方面，全面阐述了基于自然语言的知识图谱查询与可视化系统的设计与实现过程。主要工作包括：

（1）设计并实现自然语言到Cypher查询的转换模块，集成大语言模型API；

（2）设计并实现Neo4j图数据库的只读查询接口，确保数据安全；

（3）设计并实现基于ECharts的知识图谱可视化展示模块；

（4）设计并实现系统的安全校验机制，防止恶意查询；

（5）对系统进行全面的功能测试和性能评估。

\section{系统开发环境与技术选型}
\subsection{开发环境}

本系统在GNU/Linux环境下开发，采用Python作为后端开发语言，使用HTML/CSS/JavaScript实现前端界面。

\subsection{技术栈}

\subsubsection{Python 3.10+}
Python是一种解释型、高级、通用的编程语言，具有语法简洁、生态丰富的特点。本系统后端采用Python 3.10以上版本开发。

\subsubsection{FastAPI}
FastAPI是一个现代、高性能的Python Web框架，基于Starlette和Pydantic构建，支持异步编程，具有自动API文档生成、数据验证等功能，非常适合构建RESTful API服务。

\subsubsection{Neo4j 图数据库}
Neo4j是目前最流行的图数据库之一，采用属性图模型，使用Cypher查询语言，支持ACID事务，为知识图谱的存储和查询提供了高效、可靠的解决方案。

\subsubsection{大语言模型（LLM）}
系统通过调用OpenAI兼容的大语言模型API（如GPT-4、Kimi等），实现自然语言到Cypher查询的转换。利用LLM强大的语义理解能力，准确解析用户的查询意图。

\subsubsection{ECharts}
ECharts是百度开源的数据可视化库，提供了丰富的图表类型，支持大规模数据渲染，本系统使用其关系图（Graph）组件展示知识图谱。

\subsubsection{其他工具}
\begin{itemize}
    \item \textbf{Vim/Neovim}：文本编辑器，用于代码编写

    \item \textbf{httpx}：HTTP客户端，用于LLM API调用
\end{itemize}

\section{系统分析}
\subsection{可行性分析}

\begin{enumerate}[1)]
    \item \textbf{技术可行性分析}
        
        本系统所采用的技术均为成熟、稳定的技术方案。FastAPI作为现代Python Web框架，性能优异且生态完善；Neo4j作为主流图数据库，提供了稳定的Python驱动；大语言模型API服务（如OpenAI、Kimi等）已广泛应用于生产环境；ECharts是百度开源的可视化库，在数据可视化领域有着广泛应用。因此，本系统在技术层面完全可行。
    
    \item \textbf{经济可行性分析}
        
        本系统开发所使用的工具和框架均为开源免费软件，无需购买商业授权。大语言模型API调用虽然会产生一定费用，但可以通过合理设计降低调用频率，且市面上存在多种免费或低成本的API选项。总体而言，系统开发和运行成本较低，经济可行。
    
    \item \textbf{操作可行性分析}
        
        本系统采用Web界面，用户无需安装额外软件，通过浏览器即可访问。自然语言查询方式符合用户习惯，无需学习复杂的查询语法，操作简便，用户接受度高。
\end{enumerate}

\subsection{需求分析}

\subsubsection{功能需求}

（1）\textbf{自然语言查询}：用户输入自然语言描述，系统自动转换为Cypher查询并执行；

（2）\textbf{直接CQL执行}：支持用户直接输入Cypher查询语句执行；

（3）\textbf{图谱可视化}：将查询结果以关系图形式展示，支持节点拖拽、缩放等交互；

（4）\textbf{数据表格展示}：对于属性查询，以表格形式展示结果；

（5）\textbf{Schema查询}：展示知识图谱的节点标签和关系类型；

（6）\textbf{安全限制}：禁止写入操作，确保数据安全。

\subsubsection{非功能需求}

（1）\textbf{安全性}：系统只读，禁止CREATE、DELETE、SET等写操作；

（2）\textbf{性能}：查询响应时间控制在合理范围内；

（3）\textbf{可扩展性}：支持更换不同的大语言模型后端；

（4）\textbf{易用性}：界面简洁，操作直观。

\subsection{性能分析}

系统性能主要受以下因素影响：

（1）\textbf{LLM API响应时间}：自然语言转Cypher的耗时主要取决于大语言模型的推理速度，通常在1-3秒；

（2）\textbf{Neo4j查询性能}：取决于查询复杂度和数据规模，简单查询可在毫秒级完成；

（3）\textbf{前端渲染性能}：ECharts可支持万级节点流畅渲染。

通过设置查询结果上限、超时控制等机制，可保证系统的整体响应性能。

\section{系统设计}
\subsection{系统总体架构}

系统采用分层架构设计，分为表示层、业务逻辑层、数据访问层和外部服务层：

    \item \textbf{业务逻辑层}：FastAPI应用，包含API路由、业务逻辑处理、数据转换等；
    \item \textbf{数据访问层}：Neo4j客户端，负责与图数据库的交互；
    \item \textbf{外部服务层}：大语言模型API，用于自然语言到Cypher的转换。
\end{enumerate}

\subsubsection{自然语言查询模块}

该模块是系统的核心功能，主要流程如图~\ref{fig:nlq-flow}所示：

（1）用户输入自然语言查询；
（2）系统获取当前知识图谱的Schema信息（节点标签、关系类型）；
（3）构建Prompt，将Schema信息和用户查询发送给LLM；
（4）LLM生成Cypher查询语句和参数；
（5）系统进行安全校验（只读检查、EXPLAIN预检）；
（6）执行查询，获取结果；
（7）将结果转换为ECharts图数据或表格数据返回。

Prompt设计采用系统提示词+Few-shot示例的方式，引导LLM生成正确的Cypher查询。

\subsubsection{CQL执行模块}

支持用户直接输入Cypher查询语句执行：

（1）解析CQL语句，进行只读安全检查；
（2）提取查询中的参数占位符；
（3）校验必需参数是否提供；
（4）执行查询并返回结果。

\subsubsection{安全校验模块}

系统采用多层安全机制，流程如图~\ref{fig:security-flow}所示：

\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{img/security-flow.png}
    \caption{安全校验流程图}
    \label{fig:security-flow}
\end{figure}

安全机制详细说明：

（1）\textbf{黑名单检查}：使用正则表达式匹配，禁止CREATE、MERGE、DELETE、SET等写操作关键字；

（2）\textbf{EXPLAIN预检}（可选）：通过Neo4j的EXPLAIN功能预检查查询的执行计划，识别潜在的写操作；

（3）\textbf{参数校验}：确保所有参数都被正确提供，防止注入攻击；

（4）\textbf{超时限制}：设置查询执行时间上限，防止长时间查询导致系统资源耗尽。

\subsubsection{可视化模块}

（1）\textbf{图数据转换}：将Neo4j返回的节点和关系数据转换为ECharts Graph所需的格式，流程如图~\ref{fig:data-convert}所示；

（2）\textbf{节点分类}：根据节点标签设置不同的类别，使用不同颜色区分；

（3）\textbf{表格展示}：对于返回节点属性的查询，以表格形式展示详细属性信息。

\subsection{数据库设计}

本系统使用Neo4j图数据库，采用属性图模型存储数据。数据库Schema包括：

\begin{itemize}
    \item \textbf{节点（Node）}：具有标签（Label）和属性（Property）
    \item \textbf{关系（Relationship）}：具有类型（Type）、方向（Direction）和属性
\end{itemize}

具体的节点标签和关系类型根据实际应用场景定义，系统通过Schema查询接口动态获取。

\subsection{核心算法实现}

\subsubsection{自然语言转Cypher算法}

系统通过精心设计的Prompt引导大语言模型生成正确的Cypher查询。Prompt包含：

（1）\textbf{系统角色定义}：告知LLM扮演Cypher查询专家的角色；

（2）\textbf{Schema信息}：当前知识图谱的节点标签和关系类型；

（3）\textbf{Few-shot示例}：提供几个自然语言查询与对应Cypher的示例；

（4）\textbf{输出格式要求}：要求返回Cypher语句和参数的字典格式。

\subsubsection{图数据转换算法}

将Neo4j返回的记录转换为ECharts Graph格式：

（1）遍历查询结果中的每个记录；

（2）识别节点（Node）和关系（Relationship）对象；

（3）为每个节点生成唯一ID，提取标签作为类别，属性作为节点数据；

（4）为每条关系提取起始节点、目标节点和关系类型；

（5）进行节点去重，处理孤儿节点。

\subsection{前端界面设计}

前端采用简洁的单页面设计，包含以下区域：

（1）\textbf{查询输入区}：文本输入框，支持自然语言和CQL两种模式；

（2）\textbf{图谱展示区}：使用ECharts Graph展示查询结果，支持拖拽、缩放、点击查看详情；

（3）\textbf{表格展示区}：以表格形式展示节点属性；

（4）\textbf{生成的CQL展示}：显示转换后的Cypher查询语句，便于用户理解和调试。

\subsection{安全实现}

\subsubsection{只读限制实现}

使用正则表达式黑名单匹配禁止的操作：

\begin{verbatim}
WRITE_BLACKLIST = [
    r"\bCREATE\b",
    r"\bMERGE\b", 
    r"\bDELETE\b",
    r"\bDETACH\s+DELETE\b",
    r"\bSET\b",
    r"\bREMOVE\b",
    r"\bDROP\b",
    r"\bLOAD\s+CSV\b",
    r"\bCALL\s+DBMS\.",
]
\end{verbatim}

\subsubsection{参数校验实现}

使用正则表达式提取CQL中的参数占位符\texttt{\$param}，与提供的参数进行对比，确保所有必需参数都已提供。

\section*{总结}%无编号章节
\addcontentsline{toc}{section}{总结} % 添加到目录

本文设计并实现了一个基于自然语言的知识图谱查询与可视化系统。系统采用B/S架构，结合FastAPI、Neo4j、大语言模型和ECharts等技术，实现了自然语言到Cypher查询的自动转换和结果可视化。

系统的主要特点包括：
\begin{enumerate}[(1)]
    \item 降低了知识图谱查询门槛，用户无需掌握Cypher语法；
    \item 多层安全机制确保只读访问，保护数据安全；
    \item 直观的图谱可视化，便于用户理解数据关系；
    \item 模块化设计，易于扩展和维护。
\end{enumerate}

未来工作可考虑：
\begin{enumerate}[(1)]
    \item 引入用户认证和权限管理；
    \item 支持更复杂的自然语言查询（如聚合、排序）；
    \item 优化Prompt设计，提高查询转换准确率；
    \item 支持多个知识图谱数据源切换。
\end{enumerate}
